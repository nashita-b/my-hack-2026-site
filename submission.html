Submission guide

Everything your team needs to submit a project that qualifies for judging — rules, the step-by-step process, and what your Devpost entry needs to include.

Eligibility rules

What every team needs to follow.

// rules.md
  • No code before the hackathon starts — brainstorming and wireframes beforehand are fine, but your first line of code should be written after Day 1 begins.
  • Need an idea? Check out the project tracks on the home page for inspiration — not required, just a starting point.
  • Use only halal and open-source datasets.
  • Clearly cite every AI tool and API you use.
  • One team per person — you can only compete with a single team.
Deadline

Submit by 12:00 PM, Sunday July 26.

Give yourself buffer — don't wait until the last minute to upload video and links.

1

Set up your repo early

Before the hackathon starts, create a public GitHub repository named [project-name]-2026. Add your open-source license file to the root.

2

Build & document as you go

Push clean, working code throughout the event. Your README should cover the project overview, team members and roles, the problem you're solving, your tech stack, and setup instructions.

3

Record a demo video

3–5 minutes, uploaded to YouTube as Public or Unlisted. Walk through the problem, your solution, a UI demo, how you used AI, and why it matters for the Muslim community. Title it to match your repo name.

4

Deploy a live version

Get a working link up on Vercel, Netlify, Firebase, or similar (see the resources page for options). It doesn't need to be polished — it needs to be reachable and match your repo name.

5

Submit on Devpost

Go to myhack-2026.devpost.com, register, click "Enter a submission," and fill in your project title, description, team members, GitHub link, video link, live app link, and tools used. New to Devpost? watch a quick walkthrough ↗.

Before you submit

What to actually turn in.

Your Devpost entry needs all 5 of these.

  • ✓ Open-source license (Apache 2.0 or MIT) in your repo's root
  • ✓ Public GitHub repo, correctly named, with a complete README
  • ✓ Demo video (3–5 min) uploaded to YouTube, Public or Unlisted
  • ✓ Live app deployed and publicly reachable
  • ✓ Everything submitted on Devpost before the deadline
